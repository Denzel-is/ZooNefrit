# ZooNefrit — Интернет-магазин зоотоваров

Адаптивный современный сайт интернет-магазина зоотоваров. В проекте реализован удобный каталог товаров со 100 позициями, поиском, фильтрами по животным/ценам/категориям, интерактивной корзиной с расчетом стоимости доставки, модальными окнами товаров и плавными микроанимациями.

Проект полностью оптимизирован, подготовлен к публикации и настроен для быстрой локальной сборки и развертывания на любом хостинге.

---

## 🚀 Быстрый запуск

### 1. Установка зависимостей
Убедитесь, что у вас установлен [Node.js](https://nodejs.org/), затем выполните команду в корневой папке проекта:
```bash
npm install
```

### 2. Запуск в режиме разработки (Vite Dev Server)
Для быстрой локальной разработки с автоматической перезагрузкой страницы (HMR) выполните:
```bash
npm run dev
```
После запуска проект откроется по адресу [http://localhost:3000](http://localhost:3000).

### 3. Запуск локального Express-сервера (Node.js)
Если вам нужно запустить проект на Node.js сервере (имитируя production-окружение или хостинги вроде Heroku/Render/Railway):
```bash
# 1. Собрать статические файлы
npm run build

# 2. Запустить Node.js сервер
npm start
```
Сервер считывает порт из переменных окружения `process.env.PORT` или использует порт `3000` по умолчанию (`const PORT = process.env.PORT || 3000`).

## 🛠 Сборка проекта

Для компиляции, оптимизации и минификации всех HTML/CSS/JS файлов в финальный легковесный бандл выполните:
```bash
npm run build
```
Собранные готовые файлы будут находиться в папке `/dist`.

Для предварительного просмотра собранного бандла выполните:
```bash
npm run preview
```

## 🌐 Развертывание (Деплой)

Проект полностью готов для публикации. Ниже приведены пошаговые инструкции для хостинга **Netlify**.

### Вариант 1. Автоматический деплой через GitHub (Рекомендуемый)
Этот способ настраивает непрерывную интеграцию (CI/CD): при каждом вашем `git push` сайт будет автоматически пересобираться и обновляться в интернете.

1. **Создайте репозиторий на GitHub** и добавьте туда проект (это уже сделано для вашего репозитория: `https://github.com/Denzel-is/ZooNefrit.git`).
2. **Войдите в панель управления Netlify** (рекомендуется войти через ваш аккаунт GitHub).
3. Нажмите кнопку **"Add new site"** -> **"Import an existing project"**.
4. Выберите **GitHub** в качестве провайдера и предоставьте доступ к репозиторию `ZooNefrit`.
5. Netlify автоматически считает файл конфигурации `netlify.toml` из корня проекта и заполнит параметры сборки:
   - **Build command**: `npm run build`
   - **Publish directory**: `dist`
6. Нажмите **"Deploy ZooNefrit"**. Сайт опубликуется в течение минуты.

### Вариант 2. Ручной деплой (Netlify Drop)
Если вы хотите загрузить файлы без привязки к Git:

1. Выполните сборку проекта локально:
   ```bash
   npm run build
   ```
2. Перейдите на страницу [Netlify Drop](https://app.netlify.com/drop).
3. Перетащите собранную папку `dist` из корня проекта прямо в окно браузера.
4. Через несколько секунд ваш сайт будет опубликован и станет доступен по публичной ссылке.

### Вариант 3. Ручной деплой через Netlify CLI
1. Установите CLI-инструмент Netlify глобально:
   ```bash
   npm install -g netlify-cli
   ```
2. Авторизуйтесь в своем аккаунте:
   ```bash
   netlify login
   ```
3. Выполните деплой собранной папки `dist` в production:
   ```bash
   netlify deploy --dir=dist --prod
   ```

## 📂 Структура проекта

- `index.html` — главная страница интернет-магазина.
- `catalog.html` — каталог с фильтрацией и поиском товаров.
- `style.css` — стили (включая переменные темы, анимации и адаптивную верстку).
- `script.js` — логика каталога, фильтров, корзины и модального окна (подключается как модуль).
- `public/images/` — статические SVG-изображения (автоматически копируются в `/dist` при сборке).
- `server.js` — Express сервер для Node.js среды.
- `vite.config.js` — конфигурация сборщика Vite для многостраничного приложения.
- `tsconfig.json` — конфигурация TypeScript (для интеграции подсказок в редакторы).
- `netlify.toml` — файл конфигурации деплоя для Netlify.
- `.gitignore` — список игнорируемых файлов для Git.
- `LICENSE` — лицензионное соглашение проекта.
